║                            IMPLEMENTATION NOTES                           ║
╚═══════════════════════════════════════════════════════════════════════════╝

1. Prometheus Exporter Integration:
   - Embed HTTP server in mining-core crate using warp/axum
   - Use prometheus-client crate for metrics registration
   - NVML bindings via cuda-runtime-sys for GPU metrics

2. Data Retention Strategy:
   - Hot: 7 days @ 10s resolution (SSD) → 5GB/node/week
   - Warm: 30 days @ 1m resolution (HDD) → 10GB/node/month
   - Cold: 1 year @ 5m resolution (S3) → 30GB/node/year

3. Alert Routing:
   - Critical → Email + Slack + PagerDuty (immediate)
   - High → Email + Slack (15 min SLA)
   - Medium → Slack only (1 hour SLA)
   - Low → Dashboard only (next business day)

4. Security Considerations:
   - Prometheus exporter: Bind to 127.0.0.1 or private network only
   - Grafana: HTTPS + basic auth (minimum), OAuth2 (recommended)
   - Alert webhook: Use API keys/tokens for authentication
   - Firewall: Block external access to monitoring ports

5. Performance Impact:
   - Metrics collection: <1% CPU overhead
   - Exporter HTTP: <0.5% CPU, <50MB RAM
   - NVML queries: ~1ms per GPU per scrape
   - Scraping: 10s interval (configurable 5s-60s)
